# Spotit Access Control Policy

## Purpose

This policy defines how access to Spotit should be controlled before real patient use.

## Access Principles

- Access should be approved before use.
- Users should only have the access needed for their role.
- Shared accounts should not be used.
- Admin access should be limited.
- Leavers should be removed promptly.
- Access should be reviewed regularly.

## Suggested Roles

### Admin

- Approve users
- Review access requests
- Manage clinician accounts
- Review audit logs
- Remove access

### Clinician

- View assigned caseload
- Add patient details where approved
- Capture wound information
- Record assessment
- Prepare reports

### Reviewer / Buyer Demo

- Controlled access for demonstration only
- No real patient data entry

## Joiner Process

1. User requests access.
2. Admin confirms identity and role.
3. Admin creates approved account.
4. User receives login instructions.
5. User accepts local policy/training requirements.

## Leaver Process

1. Organisation notifies admin of leaver.
2. Admin disables access.
3. Audit entry is retained.
4. Organisation confirms removal.

## Review Cycle

Access should be reviewed at least monthly during pilot and before any live deployment.
