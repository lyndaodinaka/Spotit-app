This template is for discussion only and should be reviewed by legal or information governance advisers before use.

- Controller: `[Buyer / Healthcare Organisation]`
- Processor: `[Spotit / Medholic, if hosting or processing data on behalf of the Controller]`

## Subject Matter

Processing of wound-care workflow data for documentation, review planning, reporting and approved service evaluation.

## Duration

The term of the pilot, licence or service agreement, plus any agreed retention period.

## Categories of Data

- Patient identifiers
- Demographics
- Wound photographs
- Wound measurements
- Medical history relevant to wound care
- Care plans
- Reports
- Consent records
- Clinician/user account information
- Audit logs

## Special Category Data

Health data and wound images may be processed. The Controller is responsible for confirming lawful basis and Article 9 condition for processing.

## Processor Duties

The Processor should:

- Process data only on documented instructions
- Keep data confidential
- Apply appropriate technical and organisational measures
- Support data subject rights requests where applicable
- Support breach notification process
- Keep audit and access records
- Use approved subprocessors only
- Delete or return data at contract end according to Controller instruction

## Controller Duties

The Controller should:

- Confirm lawful basis
- Complete DPIA and information governance review
- Inform patients/users as required
- Approve access roles and user permissions
- Define retention rules
- Approve pilot scope and clinical use

## Security Controls

- HTTPS in transit
- Strong authentication
- Role-based access
- Audit logging
- Encrypted storage where appropriate
- Backup and restore testing
- Access removal process
- Incident response process

## Breach Notification

The Processor should notify the Controller without undue delay after becoming aware of a personal data breach.

## Deletion and Return

At the end of the agreement, data should be deleted or returned according to the Controller's written instruction and applicable retention rules.
