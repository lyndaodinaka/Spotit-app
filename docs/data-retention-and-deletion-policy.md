# Spotit Data Retention and Deletion Policy

## Purpose

This policy explains how Spotit data retention and deletion should be agreed before real patient use.

## Demo and Testing Data

Demo data should be fake or anonymised wherever possible. Demo records may be deleted when no longer needed for testing, buyer review or product demonstration.

## Real Patient Data

Real patient data must not be entered unless the deploying organisation has approved:

- Lawful basis
- DPIA
- Retention period
- Deletion process
- Patient transparency wording
- Access control process
- Backup and archive process

## Suggested Data Categories

- Access request data
- User account data
- Patient identifiers
- Wound images
- Wound assessments
- Reports
- Audit logs
- Consent records

## Retention Rules

Retention must be set by the deploying organisation. Spotit should not define clinical retention periods alone.

## Deletion Requests

Users or organisations can request deletion through:

`https://spotit-app-production-6d30.up.railway.app/delete-data.html`

Requests should be verified before deletion.

## Data That May Need Retention

Some data may need to be retained for:

- Clinical governance
- Legal claims
- Audit
- Safeguarding
- Security investigation
- Regulatory obligations

## Review Cycle

Retention rules should be reviewed at least annually or before any live pilot.
