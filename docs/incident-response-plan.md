# Spotit Incident Response Plan

## Purpose

This plan supports response to privacy, security or clinical workflow incidents involving Spotit.

## Incident Types

- Unauthorised access
- Lost or exposed login credentials
- Incorrect patient record entry
- Wound image uploaded to wrong record
- Data deletion request mishandled
- Suspected personal data breach
- Service outage
- Failed report sharing
- Clinical escalation missed

## Immediate Actions

1. Record the incident date and time.
2. Identify affected users, patients or organisations.
3. Restrict access if needed.
4. Preserve audit logs.
5. Notify the responsible organisation lead.
6. Assess whether personal data is involved.
7. Assess whether clinical risk is involved.

## Breach Assessment

The Controller should decide whether the incident is reportable to regulators or affected individuals.

## Evidence to Collect

- User account involved
- Patient record involved
- Time of event
- Audit log entries
- Screenshot if safe
- Description of what happened
- Immediate containment action
- Follow-up action

## Post-Incident Review

After resolution:

- Identify root cause
- Update process or training
- Apply technical fix where needed
- Record lessons learned
- Confirm closure with the organisation
