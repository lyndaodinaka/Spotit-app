# Spotit Launch Checklist

## Product
- Confirm product name: Spotit.
- Confirm target customers: care homes, community nursing, tissue viability services, private wound clinics, NHS teams.
- Confirm first paid product: wound assessment and dressing-change documentation.
- Confirm whether Spotit gives clinical decision support or only prompts review.

## Clinical Safety
- Appoint clinical safety lead.
- Create hazard log.
- Create clinical safety case.
- Document escalation rules.
- Make clear that Spotit supports documentation and review prompts, not diagnosis.
- Test with nurses/clinicians using realistic dummy cases.

## Regulatory And Assurance
- Check MHRA software medical device status.
- Prepare NHS DTAC evidence if selling into NHS or adult social care.
- Prepare data protection impact assessment.
- Register with ICO if required for the operating organisation.
- Prepare privacy notice, data processing agreement, user terms, and support process.

## Security
- Secure authentication.
- Role-based permissions.
- Encrypted database.
- Encrypted wound photo storage.
- Audit log for view, edit, export, send, delete.
- Backup and recovery.
- Penetration test before paid live use.
- Incident response process.

## Technical Build
- Build frontend as responsive web app/PWA.
- Build backend API.
- Build PostgreSQL database.
- Build private wound image storage.
- Build PDF report export.
- Build email/NHS mail sending workflow.
- Build admin user management.
- Build tenant/organisation setup for each customer.

## Sales Assets
- One-page product overview.
- Demo script.
- Screenshots on phone, tablet, desktop.
- Pricing model.
- Pilot proposal.
- Data protection and security summary.
- Clinical safety summary.

## First Sellable Pilot Criteria
- No real patient data until security and governance are complete.
- Pilot agreement signed.
- Named clinical lead at pilot site.
- Staff training completed.
- Support contact available.
- Clear go-live and rollback process.
